# Implementation Plan: Reference Voice Capture

**Branch**: `feature/time-machine-reference-voice-capture` | **Date**: 2026-07-15 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/003-reference-voice-capture/spec.md`

## Summary

Harden the existing reference-capture workflow so recordings and imported files are prepared exclusively into a staging WAV, validated, and only then committed as the accepted app-owned reference. Preserve the previous accepted reference and all derived state when replacement fails. Prevent overlapping preparation from the pipeline and UI, surface importer failures, and retain the existing realtime meter and best-effort on-device transcription behavior.

## Technical Context

**Language/Version**: Swift 6.0 language mode with Xcode 26 toolchain
**Primary Dependencies**: SwiftUI, Observation, AVFoundation, Accelerate, Speech, UniformTypeIdentifiers
**Storage**: App-owned Application Support session directory plus temporary prepared-reference staging file
**Testing**: Swift Testing through `MacVoiceCloningTests`, source-contract tempfile checks, and manual microphone acceptance
**Target Platform**: macOS 26.0+, Apple Silicon
**Project Type**: Native macOS application generated by XcodeGen
**Performance Goals**: Poll meter near 30 Hz without blocking the main actor; convert audio off the main actor; prevent duplicate preparation
**Constraints**: Three-second minimum; 24 kHz mono Float32 WAV; security-scoped imports; transcription is best effort; previous reference survives failed replacement
**Scale/Scope**: One active recorder, one preparation, one accepted reference, and one transcription task per application session

## Constitution Check

### Before Design

- **Local-first**: Audio preparation and speech analysis remain local; only system speech assets may be installed.
- **Privacy**: External file access is scoped to the active import, and accepted audio lives in app-owned storage.
- **Correctness**: Staging prevents partial conversion from replacing the accepted reference.
- **Concurrency**: Heavy conversion and analysis remain off the main actor; UI state stays actor-isolated.
- **User recovery**: Recording, conversion, and importer errors remain visible without destroying prior valid work.
- **Testing**: Existing conversion tests are extended and source contracts cover orchestration that depends on microphone and SpeechAnalyzer.

**Gate**: PASS

### After Design

- No network upload, credential, or new entitlement is introduced.
- The replacement commit is same-volume and occurs only after validation.
- Preparation overlap is guarded in both orchestration and controls.
- Failed replacements preserve reference, transcript, synthesis, and alteration.
- Stale transcription results remain canceled before they can mutate current state.

**Gate**: PASS

## Design Decisions

### 1. Fixed staging file with exclusive preparation

Add an app-owned staging URL because `PipelineState` serializes reference preparation. Convert to staging, validate duration, then replace or move it to the accepted reference path. Always remove leftover staging in `defer`.

### 2. Preserve the previous reference on failure

Do not clear accepted reference or downstream state before conversion succeeds. A short, unreadable, or failed replacement only updates `lastError`. Commit success then updates reference metadata, invalidates prior synthesis, and starts new transcription.

### 3. Guard preparation at two layers

`PipelineState.setReference` returns while preparation is active, which protects non-UI callers. Record and Import controls are disabled during preparation to give immediate user feedback and prevent raw-file mutation while conversion reads.

### 4. Keep transcription best effort and generation-safe

Continue canceling the previous transcription task when a valid new reference is committed. The canceled task checks cancellation before mutating transcript state; user-entered non-empty text remains authoritative.

### 5. Extend existing audio tests

Add empty-input rejection to `AudioConvertingTests`. Existing tests already prove mono 24 kHz normalization and readable duration without requiring microphone permission or speech assets.

## Implementation Sequence

1. Validate the current recording, conversion, UI, and transcription contracts.
2. Add the staging path in `SessionFiles`.
3. Guard and stage `PipelineState.setReference`, preserving old state on failure.
4. Disable conflicting controls and report importer failures in `RecordStageView`.
5. Add focused empty-input conversion coverage.
6. Run source-contract verification, XCodeMCP build, CLI build, and focused conversion tests.
7. Refresh persistent Codebase Memory.

## Complexity Tracking

| Concern | Why Needed | Simpler Alternative Rejected |
|---|---|---|
| Staging plus replacement | Prevents failed conversion from destroying accepted audio | Direct conversion currently removes the accepted destination first |
| Dual preparation guard | Protects both UI and programmatic callers | UI-only disabling leaves the model vulnerable |
| Source-contract verifier | Microphone and SpeechAnalyzer flows are environment-dependent | Full automation would require invasive dependency injection outside this feature |
