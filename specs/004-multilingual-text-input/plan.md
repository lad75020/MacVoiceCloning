# Implementation Plan: Multilingual Text Input

**Branch**: `feature/time-machine-multilingual-text-input` | **Date**: 2026-07-15 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/004-multilingual-text-input/spec.md`

## Summary

Make Auto-detect the default synthesis language, retain the exact Qwen3-TTS language catalog, normalize only boundary whitespace at request construction, and invalidate generated state whenever text or language changes. Lock stage-2 controls while generation is active so the visible inputs cannot diverge from the in-flight request. Add focused tests for language raw values, display names, locale mappings, and request defaults.

## Technical Context

**Language/Version**: Swift 6.0 language mode with Xcode 26 toolchain
**Primary Dependencies**: SwiftUI, Observation, Foundation, Qwen3-TTS through the existing engine boundary
**Storage**: In-memory observable text and language state; no persistence
**Testing**: Swift Testing for pure language/request contracts plus tempfile source-contract verification for pipeline and SwiftUI integration
**Target Platform**: macOS 26.0+, Apple Silicon
**Project Type**: Native macOS application generated by XcodeGen
**Performance Goals**: Immediate input editing and constant-time invalidation of metadata references
**Constraints**: Preserve arbitrary Unicode; no silent truncation; exact backend raw values; Auto-detect default; controls locked during non-cancellable generation
**Scale/Scope**: One target text value and one selected language per application session

## Constitution Check

### Before Design

- **Local-first**: No network language service or telemetry is introduced.
- **Privacy**: Target text stays in process and is sent only to the local model.
- **Correctness**: Generated audio cannot remain associated with changed inputs.
- **Accessibility**: Text and language controls keep explicit accessible labels.
- **Concurrency**: Controls remain immutable while the current backend generation runs.
- **Testing**: Pure value contracts are covered without loading the model.

**Gate**: PASS

### After Design

- The model-compatible language catalog is verified against the pinned package.
- Auto-detect is represented by the backend's exact `auto` raw value.
- Input observers invalidate only when values actually change.
- Reference audio and transcript remain untouched by stage-2 edits.
- No detector, persistence layer, or token estimator is added.

**Gate**: PASS

## Design Decisions

### 1. Auto-detect is the default

Set both pipeline and request defaults to `.auto`. This is the explicit clarification outcome and matches the backend default.

### 2. Preserve the exact backend language catalog

Keep eleven cases in deterministic picker order: Auto-detect, English, Chinese, Japanese, Korean, German, French, Russian, Portuguese, Spanish, and Italian. Raw values remain lowercase full names because Qwen3-TTS maps those identifiers directly to codec language IDs.

### 3. Normalize only at the synthesis boundary

Keep the user's editable value unchanged. Expose a trimmed synthesis value for readiness and request construction so whitespace-only input is invalid while intentional internal formatting remains intact.

### 4. Invalidate derived output on actual changes

Property observers compare old and new values before clearing synthesis, statistics, altered audio, and progress. Text edits do not change language; language edits do not change reference audio or transcript.

### 5. Lock inputs during synthesis

Disable the text editor and picker while `isSynthesizing` is true. The existing backend call is synchronous on its serial executor and cannot be safely canceled mid-generation, so locking prevents stale output from being presented under changed inputs.

### 6. Test pure contracts without model loading

Verify case order/raw values, display names, locale identifiers, and the `SynthesisRequest` default using Swift Testing. Verify pipeline observers and view disabling with a cleaned temporary source-contract script.

## Implementation Sequence

1. Establish a baseline for catalog, defaults, readiness, and stage controls.
2. Make pipeline language default Auto-detect and add trimmed target text.
3. Invalidate derived state when target text or language actually changes.
4. Construct requests from captured normalized text and selected language.
5. Update placeholder and control locking in `TextStageView`.
6. Add focused language/request tests and regenerate the ignored Xcode project.
7. Run XCodeMCP build, CLI build, focused tests, and source-contract verification.
8. Refresh persistent Codebase Memory.

## Complexity Tracking

| Concern | Why Needed | Simpler Alternative Rejected |
|---|---|---|
| Property observers | Centralizes correctness for UI and programmatic edits | View-only callbacks miss non-UI callers |
| Input lock during synthesis | Backend generation cannot be safely canceled | Allowing edits can label old audio with new input |
| Separate normalized value | Preserves editable text while rejecting boundary whitespace | Mutating the field on every synthesis is surprising |
